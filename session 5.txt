Task 1: Leaky ReLU Activation Function

    import numpy as np

    def leaky_relu(x, alpha=0.01):
        return np.where(x > 0, x, alpha * x)

    values = np.arange(-5, 6)
    outputs = leaky_relu(values)

    print("Input values:", values)
    print("Leaky ReLU outputs:", outputs)

Explanation: Leaky ReLU returns the original value for positive inputs.
For negative inputs, it returns alpha multiplied by the input. This
allows a small gradient for negative values and helps reduce the dying
ReLU problem.

Task 2: Parametric ReLU and Comparison Plot

    import numpy as np
    import matplotlib.pyplot as plt

    def relu(x):
        return np.maximum(0, x)

    def leaky_relu(x, alpha=0.01):
        return np.where(x > 0, x, alpha * x)

    def parametric_relu(x, a):
        return np.where(x > 0, x, a * x)

    x = np.linspace(-5, 5, 200)

    relu_output = relu(x)
    leaky_output = leaky_relu(x)
    prelu_output = parametric_relu(x, 0.2)

    plt.plot(x, relu_output, label="ReLU")
    plt.plot(x, leaky_output, label="Leaky ReLU")
    plt.plot(x, prelu_output, label="Parametric ReLU (a=0.2)")

    plt.xlabel("Input")
    plt.ylabel("Output")
    plt.title("Comparison of ReLU Activation Functions")
    plt.legend()
    plt.grid(True)
    plt.show()

Explanation: Standard ReLU gives zero for all negative inputs. Leaky
ReLU uses a small fixed slope for negative values, while Parametric ReLU
uses a parameter that can be learned during neural network training.

Task 3: Softmax Function

    import numpy as np

    def softmax(scores):
        scores = np.array(scores)
        exp_scores = np.exp(scores - np.max(scores))
        return exp_scores / np.sum(exp_scores)

    scores = [2.0, 1.0, 0.1, 0.5]

    probabilities = softmax(scores)

    print("Softmax Probabilities:")
    print(np.round(probabilities, 3))

The probabilities are approximately:

[0.574, 0.211, 0.086, 0.128]

Explanation: Softmax converts raw class scores into probabilities. All
probabilities are between 0 and 1, and their total is equal to 1.

Task 4: Hard Sigmoid Activation Function

    def hard_sigmoid(x):
        return max(0, min(1, 0.2 * x + 0.5))

    test_inputs = [-3, -1, 0, 1, 3]

    for x in test_inputs:
        print("Input:", x, "Output:", hard_sigmoid(x))

Expected results:

Input: -3 Output: 0 Input: -1 Output: 0.3 Input: 0 Output: 0.5 Input: 1
Output: 0.7 Input: 3 Output: 1

Explanation: Hard sigmoid is a simpler approximation of the sigmoid
activation function. Its output is restricted between 0 and 1.

Task 5: Choosing Activation Functions for a Food Image Classifier

(a) Hidden Dense Layer: ReLU

ReLU is a good choice for hidden dense layers because it is
computationally efficient and helps neural networks learn nonlinear
patterns while reducing the vanishing gradient problem.

(b) Output Layer with 5 Classes: Softmax

Softmax is suitable because it converts the five output scores into a
probability distribution. The probabilities sum to 1, making it useful
for mutually exclusive multi-class classification.

(c) Output Layer with Binary Classification: Sigmoid

Sigmoid is suitable for binary classification because it produces an
output between 0 and 1, which can be interpreted as the probability of
the positive class.
